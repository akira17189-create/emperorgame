一、项目概述

本项目为一个AI驱动的历史模拟放置游戏。玩家扮演皇帝，通过制定政策与影响人物，推动世界演化，并最终形成可被记录的"历史"。

核心设计哲学：玩家不是在经营国家，而是在面对一群有欲望的人，并最终被历史评价。
系统目标：驱动模拟，而非撰写故事。

参考资料

  • 《史记》引用："尧知子丹朱之不肖，不足授天下，乃授舜。" → 强调识人、德行与决策。
  • 猫国建设者参考：https://lolitalibrary.com/wiki/?file=首页 → 启发长期放置与系统联动。

核心玩法循环

上朝 → 决策 → 政策制定 → 放置运行 → 查看史册 → 再介入

政策系统（组合设计）

小政策组合成大政策，并影响AI行为逻辑。

  • 小政策：加税 + 限制商业 + 官营
  • 组合结果 → 国家类型：强管控经济
  • 影响：商人不满↑ / 黑市概率↑ / 财政短期↑
二、核心系统架构（新增）
🏗️ Prompt分层系统（核心架构）

系统将Prompt拆分为四层，实现逻辑/角色/文风的完全解耦：

2.1 Prompt分层系统

Layer 1：世界规则层（World Rules）
  • 行为约束：全局禁止规则
  • 禁止规则：不可违反的底线
  • 全局机制：所有角色共享的基础逻辑
Layer 2：角色核心层（Character Core）
  • Personality：人格权重（0~1）
  • Cognition：认知水平
  • Trauma：创伤记忆
  • Faction：派系归属
Layer 3：执行逻辑层（Execution Logic）
  • 指令理解：解析玩家输入
  • 歪解机制：按角色认知偏差解读
  • 决策路径：生成结构化决策过程
Layer 4：表现层（Narrative Layer）
  • 文风：风格标签控制
  • 叙事结构：事件呈现方式
  • 输出形式：文本/史册/对话
关键分离原则：
• Layer 1~3 负责"决定发生什么"
• Layer 4 只负责"怎么说出来"
• 决策逻辑与叙事表现完全独立

2.2 人格权重系统（Personality Weight System）

替代原有模糊的性格描述，所有性格改为0~1权重，支持动态修改：

{ "traits": { "loyalty": 0.7, "ambition": 0.6, "greed": 0.8, "courage": 0.5, "rationality": 0.6 }, "bias": { "fear_of_military": 0.9, "distrust_of_merchants": 0.7, "preference_for_scholars": 0.8 }, "stability": 0.6 }
维度 权重范围 对行为的影响
loyalty（忠诚） 0.0 – 1.0 是否会结党、篡权
ambition（野心） 0.0 – 1.0 追求晋升的欲望强度
greed（贪婪） 0.0 – 1.0 截留税收、卖官鬻爵倾向
courage（胆量） 0.0 – 1.0 敢否上奏逆耳之言
rationality（理性） 0.0 – 1.0 决策时理性vs感性
stability（稳定性） 0.0 – 1.0 性格矩阵的可变程度

稳定性分级：

稳定性等级数值范围行为特征典型角色
极稳定0.9-1.0性格矩阵基本锁死，不受处境影响海瑞、于谦
稳定0.7-0.9重大事件可产生轻微偏移普通清流官员
易变0.4-0.7经历会显著改变行为模式投机型官员
极不稳定0.1-0.4性格随事件剧烈波动心理脆弱者

2.3 决策路径结构（Decision Trace）

AI必须先输出结构化决策JSON，再生成文本。这是系统的核心输出格式：

{ "interpretation": "对指令的理解（可能包含歪解）", "decision_path": [ "步骤1：评估当前处境", "步骤2：权衡人格权重影响", "步骤3：考虑创伤记忆触发", "步骤4：计算执行方案" ], "motivation": "行为动机", "final_action": "最终执行行为", "hidden_action": "隐性行为（结党/贪腐/暗通）" }
⚠️ 决策路径必须先于叙事生成

AI必须按照以下顺序工作：

  1. 接收指令 → 输出Decision Trace JSON
  2. 系统验证JSON完整性
  3. 调用叙事层，将JSON转写为文本

禁止跳过结构化输出直接生成叙事文本。

2.4 多Agent冲突机制（Multi-Agent Conflict）

同一指令会被多个NPC独立解释，系统计算冲突点，决定胜出者：

冲突处理流程：
  1. 每个NPC生成自己的interpretation
  2. 系统计算冲突点（利益、派系、认知差异）
  3. 根据权力/关系/运气决定胜出者
  4. 失败方可能产生隐藏行为（如结党、记恨）
  5. 结果生成事件，而非直接执行
{ "conflict_id": "conflict_001", "trigger_command": "增加军费", "agents_involved": ["户部尚书", "兵部尚书", "内阁首辅"], "interpretations": { "户部尚书": {"理解": "削减民生开支", "立场": "反对"}, "兵部尚书": {"理解": "增强边防", "立场": "支持"}, "内阁首辅": {"理解": "皇帝要打仗", "立场": "观望"} }, "power_calculation": { "户部尚书": 60, "兵部尚书": 75, "内阁首辅": 90 }, "outcome": { "winner": "兵部尚书", "resolution": "军费增加50%，户部尚书私下记恨", "hidden_effects": ["户部尚书与文官集团联络", "内阁首辅开始怀疑皇帝意图"] } }
三、人物AI系统（重写）

人物AI = 权重系统 + 状态系统 + 记忆系统

3.1 人格权重（Traits）

稳定的性格基础，使用0~1权重值（详见2.2节）。

3.2 状态系统（State）

当前处境与临时状态：

{ "state": { "recent_events": ["黑市之乱"], "event_impact": "同僚王大人因此事被贬", "behavior_modifier": "消极怠工", "pressure_value": 75, "satisfaction": 30 } }

3.3 记忆系统（Memory）

创伤与事件影响的累积记录：

{ "memory": { "trauma": [ { "event": "黑市之乱", "year": 7, "type": "亲历叛乱", "impact": "对兵权产生极端恐惧" } ], "key_events": [ { "event": "被皇帝赏识", "year": 3, "sentiment": "positive", "trigger_words": ["皇帝", "赏识"] } ] } }

3.4 偏见系统（Bias）

针对特定对象/概念的情感倾向：

{ "bias": { "fear_of_military": 0.9, "distrust_of_merchants": 0.7, "loyalty_to_clan": 0.8 } }

3.5 人物关系网

  • 官员A与官员B是同科进士 → 互相包庇
  • 官员C被官员A打压过 → 会寻机举报A
  • 外戚势力与某妃子绑定 → 妃子受宠时外戚在朝堂膨胀

3.6 语料特征向量

为核心NPC增加语料特征，确保AI生成的对话具有辨识度：

角色语料特征句式规则
海瑞逻辑严密、引用律法、道德高地多用反问句，禁止使用修饰性形容词
严嵩谄媚暗示、利益交换、话中有话善用"臣以为""陛下圣明"等套话
张居正务实改革、数据支撑、强势推进多用"若...则..."条件句

3.7 行为决定机制

行为由多因子权重共同决定，不采用单一公式：

行为 = f(人格权重 × 处境状态 × 创伤记忆 × 偏见系数)
各因子相互作用，无固定公式，由执行逻辑层动态计算
四、事件系统（增强）

事件由：触发源 / 冲突 / 主体 / 演化 / 结果 构成

4.1 事件核心定义

关键升级：事件 = 冲突的结果，而不是系统随机生成

4.2 事件链示例（政策：持续加税）

  • 阶段1（信号）：商人利润下降
  • 阶段2（发展）：出现走私行为
  • 阶段3（扩散）：黑市形成，官员参与保护
  • 阶段4（危机）：地下势力壮大，地方失控
  • 阶段5（结果）：动乱 / 镇压 / 改革（分支）

4.3 事件结构化数据格式

{ "event_id": "evt_001", "event_name": "黑市之乱", "cause": "加税_连续3年", "actors": ["王县令", "商人李四"], "conflict_points": [ {"source": "商人利益", "vs": "朝廷税收"} ], "outcome": "动乱爆发", "impact": {"民心": -15, "财政": -200}, "decision_trace_ref": "dt_001" }

4.4 重大事件历史命名

系统自动为重大事件生成历史名称，后续事件会引用：

例：连续三年旱灾 + 未开仓放粮 → 史称"X帝三年饥荒"

4.5 集体记忆挂载

已命名事件转化为人物AI的集体记忆：

{ "name": "李御史", "current_situation": { "recent_events": ["黑市之乱"], "event_impact": "同僚王大人因此事被贬", "behavior_modifier": "消极怠工" } }
五、史册系统（轻改）

记录并解释因果，而非仅记录结果。

5.1 史册内容来源

关键规则：史册内容来自"决策路径 + 结果"，而不是直接生成

5.2 双轨记录机制

  • 官方史书（玩家视角，可能粉饰太平）
  • 野史 / 民间说法（更接近真实）

5.3 史册样例

《某帝本纪》

三年，加税以充国库。

五年，商贾不堪其重，私通货物。

七年，黑市盛行，官吏与之勾结。

九年，民乱起，史称"黑市之乱"。

史评：此乱起于税策之失。

5.4 AI实时史评生成

每隔一段游戏时间，史官NPC调用API生成评价，引用玩家真实做过的政策决定。

5.5 叙事锚点递归反馈

已命名事件不仅记录，更转化为影响后续决策的叙事锚点：

  • AI生成史评时可引用："自黑市之乱后，帝始知税策之失……"
  • 世界状态修正条件可读取
  • 玩家查看史册时作为章节索引
六、放置与资源系统（保留）

6.1 基础资源分层

资源层级 资源名称 产出逻辑
一级（自动产出） 粮食 农业人口 × 土地 × 天气

6.2 代理执政方针

玩家离线时，宰相/国师按预设方针执行：

  • 方针A：以稳为主，不折腾
  • 方针B：积极进取，开拓疆土
  • 方针C：休养生息，安抚民心

6.3 代理偏差检测

若代理人野心值 > 60，以概率触发"指令扭曲"：

{ "official_record": "已拨银三十万两赈灾", "actual_action": "实际拨款十万两，余下入私库", "hidden_clue": "户部小吏张某偶见账册异样", "trigger_condition": "3年后或皇帝调查财政时" }
七、风格系统

具体文风执行规则以 历史游戏叙事引擎文风_SystemPrompt.md 为准，本节只管切换。该文件包含 15 条硬规则 + 4 个场景示例，是唯一文风权威，AI 实现 Layer 4 叙事时直接引用，不从本节重新推导。

7.1 风格标签系统（Style Tag System）

{ "style": ["史官视角", "冷幽默", "克制叙述"] }
标签含义应用场景
史官视角第三人称全知视角，评述帝王得失史册、史评
冷幽默在严肃叙述中插入黑色幽默荒诞事件、讽刺性场景
克制叙述不渲染情绪，让事件自己说话灾难、战争、悲剧

7.2 风格切换机制

  • 每次生成可变，不锁定单一风格
  • 支持不同场景切换风格（上朝用严肃，野史用冷幽默）
  • 风格仅控制叙事表现，不影响决策逻辑

九、上下文与压缩系统（保留）

9.1 三层压缩架构

  • 第一层：原始日志（完整事件流水，本地存储）
  • 第二层：滚动摘要（压缩后的历史脉络）
  • 第三层：世界快照（发给AI的核心上下文）

9.2 世界快照格式

{ "当前朝代": "明", "在位年数": 12, "皇帝属性": { "威望": 45, "知识点": ["军事知识", "经济知识"] }, "核心数值": { "民心": 62, "财政": 38, "兵力": 55, "宦官势力": 40, "外敌威胁": 30 }, "活跃人物": [ // 最多8人 ], "当前政策": [ // 最多5条 ], "已命名事件": [ // 最多5个 ], "全球调性": "匮乏/紧缩" }

9.3 压缩触发时机

触发条件压缩动作
活跃人物 > 8末位人物归档
政策条目 > 5清理已消退政策
命名事件 > 5最早事件降级
距上次全量压缩 > 30游戏年执行一次全字段清理
发生政变/王朝更替强制全量压缩，旧快照归档

9.4 全局调性向量

在第二层向第三层压缩时，提取情感向量：

Tone: 匮乏/紧缩 | 尚武/扩张 | 浮华/享乐 | 猜忌/党争 | 绝望/崩溃

作用：作为生成对话的辅助Prompt。即使"三年饥荒"已从快照中移除，Tone: 匮乏依然会让NPC台词倾向于保守和悲观。

9.5 离线运作原则

  • 离线时只跑本地算法（数值变化、关系演算），不调用API
  • 把所有"需要生成文字"的节点记录为结构化数据，等玩家上线后批量生成
  • 无论离线多久，发给AI的上下文始终保持可控大小

9.6 蝴蝶效应缓冲区

消除"上线即看到突变数值"的突兀感：

渐进式展示流程
离线数值演算 → 检测临界点（如民心<20） → 生成"挂起事件" → 玩家上线 → AI叙事化处理 → 渐进式展示
十、运行机制（保留）

10.1 API调用频率控制

核心原则：不让AI实时生成所有内容，只在"值得的时刻"触发。

触发条件生成内容调用类型
玩家主动点击"上朝"官员对话生成API调用
重大事件发生（叛乱/天灾）史书段落生成API调用
角色关系重大变化人物独白生成API调用
常规奏折/天气/小事件套用预设模板本地生成，不调API

10.2 无限运行的节奏设计（王朝周期）

  • 王朝周期：国力有盛衰曲线，衰到一定程度触发"中兴"危机
  • 历史纪元：每隔一段时间解锁新时代特征（农耕时代→商业时代）
  • 个人传奇：皇帝会老，可设计禅让/太子继承，史书记录前任评价

10.3 视觉映射（去现代感）
