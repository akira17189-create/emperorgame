# 牢饭防毒与质量检查程序

## 触发条件

当满足以下条件时立即执行本程序：
- 差役送来牢饭或集体供食
- 需要验证食物是否被下毒
- 怀疑粮食质量存在问题（如官仓好米被倒卖，实际供应霉米、掺沙石饭食）
- 处于值房或临时签押房等可控场所

## 执行步骤

### 1. 现场分饭

在值房（或临时签押房）内，要求送饭差役将桶里的饭分入碗中。

**命令要点**：
- 必须在监督视线范围内操作
- 确保所有饭食都经过分装

### 2. 下达强制试吃命令

明确命令送饭差役：
> "每碗都吃一口"

**约束确认**：
- 确保差役理解此命令为强制性，无豁免
- 确认差役处于可执行状态（持有筷子、可进食）

### 3. 执行试吃与观察

监督差役完成以下动作：

1. **取食动作**：拿起筷子，端起每一碗分好的饭
2. **品尝动作**：挑起一团饭送入嘴里咀嚼
3. **反应观察**：密切观察差役表情和反应
   - 脸色是否发苦
   - 是否难以下咽
   - 是否有呕吐或不适反应

**质量验证指标**：
- 正常反应：顺利咀嚼吞咽，无异常表情
- 异常指标：脸苦、咀嚼困难、发现沙石声、霉味等
- **揭露问题**：通常可发现官仓好米被卖，人犯实际食用陈年霉米或掺沙石饭食

### 4. 确保全员覆盖

执行全覆盖验证：
- 确保所有分好的饭碗（如十二碗）都被尝遍
- 可采用多人分担制（如一人尝六口，两人尝完所有碗）
- **禁止遗漏**：未品尝的饭碗不得送入牢房

### 5. 宣告防毒规则

在试吃完成后、放行前，明确告知所有相关人员：
> "不要打量着在饭里下毒"

**宣告范围**：
- 现场送饭差役
- 做饭人员（如在场）
- 其他相关狱卒

### 6. 确立连带责任

制定并宣读惩罚机制，建立责任绑定：
> "毒死一个人犯，做饭的送饭的就把饭自己吃下去"

**责任链条**：
- 做饭人员 → 制作责任
- 送饭差役 → 传递责任
- 连带责任：任何环节下毒，全链条人员共同承担后果

### 7. 放行确认

确认以下全部条件后，允许将碗叠入桶中，提入牢房通道：

- [ ] 所有饭碗均已完成试吃
- [ ] 试吃人员无异常中毒反应
- [ ] 试吃人员无质量异议（或已记录异常）
- [ ] 防毒规则已明确宣告
- [ ] 连带责任机制已明确

**异常处理**：
- 若发现异常或下毒企图：立即执行连带责任规则，禁止问题饭食进入牢房
- 若试吃正常：签字确认或口头确认后放行

## 关键要素

| 要素 | 说明 |
|------|------|
| **牢饭** | 官仓配拨的牢粮，常被替换为陈年霉米或掺沙石的饭 |
| **差役** | 负责做饭和送饭的人员，需承担试吃风险与连带责任 |
| **试吃覆盖** | 必须覆盖所有分装碗，禁止抽样检查 |
| **连带责任** | 从制作到送达的全链条安全责任绑定 |

## 输出结果

- **安全确认**：差役试吃无异常且未下毒，饭食送入牢房供人犯食用
- **风险拦截**：发现异常或企图下毒，按规则由差役自行承担后果（吃下有毒饭食），并追究责任
- **质量揭露**：通过试吃反应揭露粮食质量问题（霉变、掺假等）
