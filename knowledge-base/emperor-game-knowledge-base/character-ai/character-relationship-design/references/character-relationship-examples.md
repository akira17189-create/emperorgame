# 人物关系设计示例

## 连结点类型示例

### 学术兴趣连结
- 中心人物：一位隐藏着学术造诣的商人
- 卡司成员：大学教授
- 连结点：对古典哲学的共同热爱
- 揭露效果：展现商人内心深处的知识分子面向

### 宗教情怀连结
- 中心人物：表面无神论的医生
- 卡司成员：虔诚的病人
- 连结点：对生命意义的探讨
- 揭露效果：揭示医生内心的信仰挣扎

### 浪漫情愫连结
- 中心人物：冷酷的侦探
- 卡司成员：热情的艺术家
- 连结点：对美的共同追求
- 揭露效果：展现侦探被压抑的感性一面

## 常见错误

### 错误1：全方位连结
**问题**：让两个角色在所有方面都相通
**修正**：选择特定方面建立连结，保留差异

### 错误2：功能重复
**问题**：多个卡司成员揭示同一面向
**修正**：为每个成员分配独特的揭示功能

### 错误3：过度揭露
**问题**：让中心人物过早暴露全部面貌
**修正**：渐进式揭露，保留重大秘密至高潮
