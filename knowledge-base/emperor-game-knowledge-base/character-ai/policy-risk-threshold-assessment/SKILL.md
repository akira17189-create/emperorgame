# 政策风险阈值评估

## 适用场景

在以下情况下使用本技能：

- 执行重大战略政策（如改稻为桑等农业结构转型）
- 政策执行已造成破坏性后果（如农田淹没）
- 尚未发生人员死亡，但存在人道主义风险
- 需要判定执行者是否应承担政治责任

## 核心评估框架

### 价值判定层级

1. **宏观层面**：确认政策是否属于"功在国家，利在千秋"的战略级政策
   - 战略级政策的长期价值超越短期局部损失
   - 具体执行造成的破坏属于可接受的战术成本

2. **关键阈值设定**：
   - **红线指标**：人员生存权（零死亡）
   - **可承受指标**：财产损失（农田淹没等）

### 决策树逻辑

```
IF (田淹了 = TRUE) AND (饿死人 = FALSE) THEN
    结果 = "什么都好说"（可协商解决，不构成政治罪责）
ELSE IF (饿死人 >= 1) THEN
    结果 = "罪"（构成严重政治罪责，需追究责任）
```

## 执行步骤

### 步骤1：确认政策性质

判定该政策是否属于战略级政策：
- 是否涉及国家长远利益
- 是否具有"功在国家，利在千秋"的特征
- 是否属于强制性的结构性调整

### 步骤2：评估破坏程度

检查以下变量：
- **田淹了**（布尔值）：是否发生农田淹没事件
- **饿死人**（整数）：因政策执行导致的饥荒死亡人数

### 步骤3：应用阈值判定

根据决策树逻辑进行判定：
- 若仅财产损失、无人员伤亡 → 执行者功大于过
- 若发生人员死亡 → 构成重罪

### 步骤4：责任转移机制

若判定为可接受风险，实施责任转移：
- 将粮食供应责任明确转移给承包商
- 设定刚性约束："买田的粮食要加紧抢运，饿死了一个人，那便是罪"
- 将政策执行风险转化为供应商的合同义务

## 输出结果

根据评估结果输出：
- **可接受**：未突破红线（零死亡），执行者功大于过
- **不可接受**：突破红线，构成严重政治罪责

## 约束条件

本技能不适用于：
- 非战略级地方政策
- 已造成人员死亡的事后处置
- 非强制性的农业转型项目
