# 人物关系设计原则

## 使用场景
当需要构建人物关系网、设计卡司结构，或通过角色互动揭示中心人物隐藏特征时使用此技能。

## 前置条件
- 已确定中心人物或主要角色
- 需要深化人物形象或推动剧情发展

## 核心原则
每个卡司成员都应能带出其他成员的特征与真相。在戏剧高潮时，让读者/观众认识该人物的程度超出他的自知。

## 执行步骤

### 1. 确定中心人物
选择被刻画的主要角色作为卡司构建的核心。

### 2. 构建卡司成员
为每位成员分配特定功能：设计他们去描绘中心人物的某一个面貌。

### 3. 设计连结机制
建立具体的连结点，例如：
- 共通的学术兴趣
- 相同的宗教情怀
- 萌生的浪漫情愫

### 4. 实施揭露过程
在中心人物面对不同成员的过程中，他的特征与不同面向会自然而然揭露。

## 关键约束
- 人物之间不可能在所有方面都相通
- 连结通常基于特定方面（如兴趣、宗教、情感）
- 没有人会对外揭露全部的自己，只暴露某些面貌，保留某些给自己，并藏起重大的秘密
- 没有人能让自己看到自己的全貌，只有其他人可以察觉某些真相

## 预期结果
一个功能互补的卡司结构，通过人物互动揭示隐藏特征，使读者比人物自己更了解他。
