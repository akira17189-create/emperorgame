# 00 — 项目总览

## 游戏定位

**AI 驱动的架空历史模拟放置游戏**。

- **体裁**：放置 + 模拟 + 历史叙事
- **世界观**：**架空王朝，非真实历史**。世界观全面参考明朝（制度、文化、官制、美学），但朝代名、皇帝名、NPC 均为虚构。不受真实历史时间线约束，不需要对应真实年份或真实人物。
- **设计意图**：借明朝的"骨架"（内阁制、宦官、党争、边患、税制、科举）搭建游戏世界，保留历史质感和复杂性，同时给剧情设计和 AI 生成留足自由度，避免"历史正确性"的掣肘。
- **玩家角色**：皇帝（可能为穿越者）
- **玩法核心**：下达自然语言指令 → NPC 用各自人格解读（可能歪解） → 事件连锁演化 → 进入史册 → 继续演化
- **美学基调**：宣纸底色、半文半白、参考《明朝那些事儿》

## 核心玩法循环

```
上朝 → 决策 → 政策制定 → 放置运行 → 查看史册 → 再介入
```

## 设计哲学（V3 §1 原文保留）

> 玩家不是在经营国家，而是在面对一群有欲望的人，并最终被历史评价。
>
> 系统目标：**驱动模拟，而非撰写故事**。

## 关键术语表

| 术语 | 含义 |
|------|------|
| **Layer 1 世界规则** | 全局禁止与物理法则，所有 NPC 共享 |
| **Layer 2 角色核心** | 单个 NPC 的人格权重 / 记忆 / 偏见 |
| **Layer 3 执行逻辑** | 指令解析、歪解、决策路径生成 |
| **Layer 4 叙事表现** | 把结构化决策翻译为中文文本，仅受文风 15 条约束 |
| **Agent / NPC** | 一个独立决策实体（海瑞、严嵩、张居正…） |
| **Decision Trace** | AI 输出的结构化决策 JSON，必须先于叙事生成 |
| **世界快照** | 每次 API 调用携带的核心上下文（≤ 500 token） |
| **全局调性** | 5 选 1 的情感向量（匮乏/尚武/浮华/猜忌/绝望），影响叙事 |
| **技能路由** | 按事件类型查字典，动态选 2~4 个子技能拼进 Prompt |
| **挂起事件** | 离线演算触发临界点生成的"待叙述"事件，上线后渲染 |
| **三档生成** | A=完整 API / B=便宜 API / C=本地模板，控制成本 |
| **自填 LLM 配置** | 玩家在设置页填 baseURL / model / apiKey，存 localStorage |
