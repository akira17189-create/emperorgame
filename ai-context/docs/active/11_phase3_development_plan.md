## 3. Phase 3.1：政策系统详细设计

### 3.1.1 设计理念
- **真实反馈**：每条政策对三层资源、NPC立场、王朝长期趋势产生连锁影响
- **冲突机制**：政策触发NPC冲突，需要皇帝仲裁
- **长期效应**：政策有短期成本和长期收益，形成策略深度
- **沉浸叙事**：政策颁布后生成朝堂反应和民间反馈叙事

### 3.1.2 与现有引擎衔接
```typescript
// 政策生效流程
玩家颁布政策 → Policy Interpreter解析 → 生成Policy对象 → 
→ 影响三层资源计算 → 触发NPC冲突/支持 → 仲裁系统介入 → 
→ 生成政策效果叙事 → 史册记录 → Tick期间持续生效
```

### 3.1.3 政策分类（四大类22个）
| 类别 | 数量 | 设计重点 | 示例政策 |
|------|------|---------|----------|
| **内政** | 6个 | 民生、吏治、税收 | 均田令、科举改革 |
| **军事** | 5个 | 边防、军制、装备 | 边军改制、火器营 |
| **仙道** | 6个 | 修道、炼丹、风水 | 炼丹术推广、观星台 |
| **工业** | 5个 | 技术、贸易、基建 | 蒸汽机研发、官道修建 |

### 3.1.4 验收标准
- ✅ 玩家能在朝堂「颁布政策」界面选择或自定义政策
- ✅ Tick时能看到资源数值变化 + NPC反应
- ✅ 政策冲突自动触发仲裁系统
- ✅ 政策效果在史册中详细记录

## 4. Phase 3.2：世界事件系统详细设计

### 4.2.1 设计理念
- **动态世界**：事件让游戏世界「活」起来，不只是静态资源变化
- **多结局分支**：每个事件有3-5种不同结局，取决于玩家决策
- **概率+策略触发**：既有随机事件，也有资源阈值触发事件
- **叙事融合**：事件无缝融入现有叙事引擎

### 4.2.2 与现有引擎衔接
```typescript
// 事件触发流程
Tick检查触发条件 → 事件触发判断 → 生成事件对象 → 
→ 呈现事件描述和选项 → 玩家决策 → 执行事件效果 → 
→ 影响资源/NPC → 生成事件结局叙事 → 史册记录
```

### 4.2.3 事件分类（六大类32个）
| 类别 | 数量 | 触发条件 | 设计重点 |
|------|------|---------|----------|
| **自然灾害** | 5个 | 概率触发 | 水灾、旱灾、地震 |
| **仙道异变** | 6个 | 仙道资源阈值 | 灵气潮汐、异兽现世 |
| **工业革命事件** | 5个 | 工业资源阈值 | 技术突破、劳工抗议 |
| **党争/宦官作乱** | 6个 | 政治派系关系 | 科举舞弊、东林党争 |
| **边疆危机** | 5个 | 军事/威胁阈值 | 倭寇入侵、边军哗变 |
| **民间异动** | 5个 | 民生资源阈值 | 饥民暴动、商帮崛起 |

### 4.2.4 验收标准
- ✅ 放置期间能随机/条件触发事件
- ✅ 玩家能看到事件叙事 + 需要做出决策
- ✅ 事件后果影响游戏状态
- ✅ 事件在史册中完整记录

## 5. Phase 3.3：UI/UX优化详细设计

### 5.3.1 设计理念
- **流畅体验**：消除卡顿，优化响应速度
- **沉浸反馈**：让玩家「感受」到决策的影响力
- **信息清晰**：关键数据一目了然，避免信息过载
- **操作便捷**：减少点击次数，提升操作效率

### 5.3.2 优化重点
1. **性能优化**
   - 虚拟列表处理长史册
   - 节流/防抖处理高频操作
   - 添加Loading状态反馈

2. **界面美化**
   - 优化CourtPage布局（政策面板、资源仪表盘）
   - 美化ChroniclePage时间线
   - 统一视觉设计语言

3. **功能增强**
   - 快速存档/读档功能
   - 一键查看政策影响
   - 决策历史回溯

### 5.3.3 验收标准
- ✅ 界面流畅，无明显卡顿（60fps）
- ✅ 沉浸感强，决策反馈即时
- ✅ 关键信息一目了然
- ✅ 操作便捷，新手也能快速上手

## 7. 风险控制与技术考量

### 7.1 Token预算控制
- 政策/事件系统会增加LLM调用频率
- 解决方案：在`llm.ts`增加「廉价模型优先 + 缓存」逻辑
- 事件叙事采用模板化 + 关键变量填充，减少Token消耗

### 7.2 数值平衡
- 政策效果需经过多轮测试调整
- 建立「政策影响计算器」预测长期效应
- 避免单条政策过强，保持策略多样性

### 7.3 向后兼容
- 所有新增功能必须与现有存档兼容
- 新增字段提供默认值初始化
- 旧存档升级时自动适配新系统
