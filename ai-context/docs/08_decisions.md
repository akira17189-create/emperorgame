## 决策列表

### D1 · 后端路线：纯静态，不部署后端
- **日期**：2026-04-17
- **决策**：游戏托管在 GitHub Pages，不设自建后端、不设代理服务器。
- **理由**：部署成本 = 0，合规成本 = 0，API key 由玩家自管，迭代速度最快。
- **影响文件**：`01_architecture.md §1`

### D2 · LLM 接入：玩家自填，不设默认提供方
- **日期**：2026-04-17
- **决策**：游戏设置页提供 `baseURL / model / apiKey` 三个字段，玩家填入任意 OpenAI 兼容服务；配置存 localStorage，不上云。
- **理由**：不绑定任何提供商，用户自主控制成本，仓库不持有任何密钥。
- **影响文件**：`01_architecture.md §2`

### D3 · 文风权威：以独立 MD 为唯一来源
- **日期**：2026-04-17
- **决策**：叙事文风规则以 `../历史游戏叙事引擎文风_SystemPrompt.md` 的 15 条为准，V3 §7 只做指针，不重复维护规则内容。
- **理由**：防止两份文档内容分叉导致 AI 执行时困惑。
- **影响文件**：V3 HTML §7（已做最小修订）、`05_prompt_layers.md §4`

### D4 · MVP 范围：只做"上朝 + 史册"垂直切片
- **日期**：2026-04-17
- **决策**：阶段 1 只实现 LLM 配置 → 新建档 → 金銮殿指令 → 决策 JSON → 叙事 → 史册写入 → IDB 存档 这一主线，其他功能后续扩展。
- **理由**：先打通 end-to-end 链路，验证架构可行再扩展。
- **影响文件**：`06_mvp_scope.md`

### D5 · 云存档：留 hook，MVP 不实装
- **日期**：2026-04-17
- **决策**：`SaveAdapter` 接口在 MVP 就要存在；`IDBAdapter` 实装；`SupabaseAdapter` 只写空类 + TODO。不在 MVP 阶段引入 Supabase 依赖。
- **理由**：MVP 优先验证玩法，云存档是基础设施，晚上线成本低。
- **影响文件**：`01_architecture.md §6`、`06_mvp_scope.md §4`

### D6 · V3 问题清单：独立文件，不污染原文
- **日期**：2026-04-17
- **决策**：V3 HTML 是只读参考文档，评审意见放 `02_review_v3.md`，不直接编辑 V3 原文（除 D7 授权的最小修订）。
- **理由**：保持原始文档作为"历史版本"，评审与原文不混淆。
- **影响文件**：`02_review_v3.md`

### D7 · V3 §7 最小修订：消除文风歧义
- **日期**：2026-04-17
- **决策**：对 V3 HTML 只改 §7「风格系统」：
  - 删除误导性的"已删除 70%"warning
  - 删除 §7.3 "保留的核心文风原则（仅3条）"（与独立 MD 重复）
  - 新增一行指针，指向 `历史游戏叙事引擎文风_SystemPrompt.md`
  - 不动其他章节
- **理由**：消除 AI 可能产生的"3 标签"与"15 条"之间的歧义。
- **影响文件**：`AI_History_Game_Design_V3_Executable.html §7`

### D8 · P1 问题：本方案文档中一次性填充
- **日期**：2026-04-17
- **决策**：V3 中标记为 P1（高优先级但未填充）的内容，在 `03_fixes_p1.md` 中给出可执行的具体内容（世界规则 20 条、多 Agent 调度、记忆 LRU、离线公式、skill 路由整合等）。
- **理由**：让 AI 编码时有直接可抄的来源，不再需要临时推断或请求用户补充。
- **影响文件**：`03_fixes_p1.md`

### D9 · 本阶段产物：只写文档，不写业务代码
- **日期**：2026-04-17
- **决策**：当前工作（生成 `docs/` 目录 + 修订 V3 §7）不包含任何业务代码、依赖安装、API 调用。
- **理由**：先让方案对齐，再动手编码，避免在未确认的设计上浪费实现成本。
- **下一步**：`07_preparation.md` 的 Ready 标准满足后，进入 MVP 编码阶段。

### D10 · 世界观：架空王朝，全面参考明朝
- **日期**：2026-04-17
- **决策**：游戏世界是虚构架空王朝，不设定在真实历史时间线上。朝代名、皇帝名、NPC 均为虚构人物。世界观全面参考明朝的制度、官制、文化、美学、党争模式，但不绑定真实年份和真实人物。
- **理由**：避免"历史正确性"约束（不用担心某人物彼年龄几岁、某事件是否已发生），给剧情和 AI 生成留足自由度；同时保留明朝的丰富历史质感作为设计素材。
- **对内容设计的影响**：NPC 设定可以"原型参考"历史人物（如原型=海瑞），但名字、年龄、具体职务均为虚构；种子剧本写"架空·开国第N年"而非"明朝·嘉靖X年"。
- **影响文件**：`00_overview.md`、`06_mvp_scope.md`（种子剧本描述）、给 mimo 的设计 prompt
