## 一、现状问题

### 开场问题

当前开场（`CourtPage.tsx` 中的 `NEWBIE_GUIDE_NARRATIVE`）存在以下问题：

1. **视角错位**：开场直接切入"皇帝坐在乾清宫等待旨意"，跳过了玩家作为穿越者"意识到自己在哪里"的关键时刻，缺少穿越觉醒的内心戏。
2. **引导说明文过于出戏**：大段 `**第一件事**`/`**第二件事**` 的游戏机制说明以教程形式插入叙事，打破沉浸感。
3. **国师玄明缺席**：游戏设定中国师是"第一引导人"，但当前开场中国师完全不在场。
4. **执行面板无入口**：玩家进入游戏后直接面对空白输入框，没有结构化的"上朝/去哪里/做什么"选项入口。
5. **逻辑错误**：`condition: game_year === 1 && chronicle.length === 0` 判断新手引导，混淆了"首次开场"和"第一年开局"的概念。

### 放置系统缺失问题

当前系统**完全没有**实时被动积累逻辑：

- 无 `setInterval` / 实时计时器
- 无离线时间补算
- 资源（food / fiscal / commerce 等）**只在 LLM gameTick 调用时**被修改
- `real_time_played_ms` 字段仅作为统计存在，未驱动任何游戏逻辑
- 游戏体验是纯回合制，玩家不主动操作时世界静止，与"放置游戏"定位不符

## 二、目标状态

### 开场流程

```
[阶段一] 穿越内心戏（纯叙事，无交互）
        ↓
[阶段二] 国师登场 + 引导对话（带选项，限定范围）
        ↓
[阶段三] 执行面板解锁（正式游戏入口）
        ↓
[正式游戏循环] 放置积累持续运行 + 玩家主动 tick 推进剧情
```

### 放置系统目标

参考 Kittens Game 的核心设计原则：
- 资源**随真实时间流逝被动积累**，玩家不操作时游戏仍在运转
- **离线补算**：玩家关掉页面再回来，按实际离开时长补发积累量
- 积累速率由**现有政策和资源状态**决定，不引入新的建筑系统（保持现有架构）
- 积累为**小额持续流入**，不替代 LLM tick——LLM tick 负责剧情推进和重大状态变化，放置系统负责基础资源的"生命迹象感"

## 三、开场三阶段规格

### 阶段一：穿越内心戏

**触发条件**：`meta.game_year === 1 && meta.prologue_phase === 'awakening'`

**场景设定（提供给 DeepSeek 的创作基础大纲）**：
- 地点：御花园，非金銮殿、非乾清宫（玩家不是在处理政务，而是在"放空"，符合国师随后出场的设定——国师选择在皇帝消遣时靠近）
- 时间：清晨或午后，阳光气候由世界观基础设定决定（承德元年，靖朝第五代，国库空虚但天气尚可）
- 内心戏内容须包含：
  - 意识到身体/环境异常（语言、感觉、周围人的称呼）
  - 搜索记忆：意识到原来的"自己"已死，灵魂穿越进来
  - 短暂的恐慌或茫然
  - 尝试用现代认知框架理解当前处境（这是玩家角色的核心人设：理性科学思维、无神论者）
  - 落脚点：暂时接受现实，决定先观察

**文案要求**（交给 DeepSeek）：
- 第一人称内心独白 + 第三人称场景描写混用
- 不超过 400 字
- 不出现任何游戏机制说明词（Tick、政策、面板等）
- 末尾留白，等待"有人靠近"的感知

**渲染方式**：打字机效果（复用现有 `typewriterEffect`），无选项，自动推进至阶段二

---

### 阶段二：国师登场与引导对话

**触发条件**：阶段一完成后自动触发，`prologue_phase` 切换至 `'guoshi_intro'`

**国师人设约束**（直接来自设定文档，提供给 DeepSeek）：

| 维度 | 内容 |
|------|------|
| 身份 | 国师玄明，18岁女性伪装男性，道士打扮 |
| 人格状态 | 初次相见触发"天真的道士"人格——日常好奇状态 |
| 对皇帝的感知 | 察觉到皇帝"变了"，但说辞模糊，不挑明（"气息有异"/"眼神陌生"之类，不说"你是穿越者"） |
| 说话风格 | 神神叨叨，喜欢用反问和暗语，不给直接答案 |
| 目的 | 表面上是来"观星报告"，实际上在试探新皇帝 |
| 禁忌 | 不主动暴露女性身份；不说"穿越"；不直接解释游戏机制 |

**对话结构（大纲，由 DeepSeek 填充文案）**：

```
国师出场：御花园，礼节性问安，但措辞暧昧（符合设定中的"贝阿朵莉切式"）
  ↓
国师旁白/单句：察觉皇帝"气息有变"，但以星象、天命等措辞模糊表达
  ↓
玩家选项 A / B / C（见下方）
  ↓
国师根据玩家选择给出不同的一句回应（不展开，保持神秘）
  ↓
国师提示：陛下今日是否要临朝，或另有安排？（自然引出阶段三）
```

**阶段二选项设计**：

选项不超过 3 个，不影响数值，目的是让玩家感受到"穿越者视角"：

| 选项 | 内心独白（括号内，仅玩家可见） | 显示文字 |
|------|-------------------------------|----------|
| A | （装作什么都没发生） | "无事，朕只是在想些事情。" |
| B | （试探这个神秘的人） | "你觉得朕今日……有何不同？" |
| C | （沉默，打量对方） | [沉默，静静打量] |

三个选项都推进至阶段三，仅国师回应文字不同（由 DeepSeek 写三条）。选项 B 的回应相对更意味深长，为后续关系线做铺垫，但不产生数值差异。

**渲染方式**：对话气泡样式或现有叙事框 + 选项按钮（复用 `event-engine` 中的 `EventChoice` 结构）

## 四、放置积累系统

### 4.1 设计原则

| 原则 | 说明 |
|------|------|
| **双轨并行** | 放置积累（实时微增）与 LLM gameTick（年度宏观推进）完全独立，互不替代 |
| **轻量嵌入** | 不引入新建筑系统，积累速率由现有 `resources` 和 `policies.active` 决定 |
| **可感知** | 玩家能看到资源数字在缓慢跳动，感受到"帝国在运转" |
| **离线补算** | 关页签再回来，按实际离开时长补发，上限 24 小时 |
| **不破坏叙事** | 放置积累只动基础数字，不生成叙事文本、不触发事件 |

### 4.2 参与放置积累的资源

| 资源字段 | 方向 | 逻辑依据 |
|----------|------|----------|
| `food` | 积累（受 `agri_pop`、`land_fertility` 影响） | 农业人口每天产粮 |
| `fiscal` | 积累（受 `tax_rate`、`commerce` 影响） | 税收和商业每天流入国库 |
| `commerce` | 缓慢积累（基准值，受政策加成） | 市场自然增长 |
| `military` | 消耗（受 `military_cost` 影响） | 军队每天消耗军饷 |
| `population` | 极缓慢积累（固定低速率） | 自然人口增长 |
| `morale` | **不参与** | 由 LLM 叙事驱动 |
| `threat` | **不参与** | 由 LLM 叙事驱动 |
| `faction` | **不参与** | 由 LLM 叙事驱动 |
| `eunuch` | **不参与** | 由 LLM 叙事驱动 |

### 4.3 积累速率公式

以**每分钟真实时间**为基准单位。系数统一放在 `idle-config.ts`，便于平衡调整：

```typescript
// 粮食积累（每分钟）
food_rate = agri_pop * land_fertility * BASE_FOOD_RATE
           + policy_bonus('agriculture')

// 财政积累（每分钟）
fiscal_rate = commerce * tax_rate * BASE_FISCAL_RATE
             + policy_bonus('taxation')

// 军事消耗（每分钟，负值）
military_drain = military_cost * BASE_MILITARY_DRAIN_RATE

// 商业缓增（每分钟）
commerce_rate = commerce * BASE_COMMERCE_GROWTH

// 人口微增（每分钟）
population_rate = population * BASE_POPULATION_GROWTH
```

**上下限保护**：
- `food`、`fiscal`、`military` 不低于 0
- 军费耗尽时 `military` 停止下降，不进入负数（负数影响留给 LLM tick 的事件系统处理）

### 4.4 实时积累实现

在 `CourtPage.tsx` 的 `useEffect` 中使用 `setInterval`，每 10 秒执行一次放置积累（对应约 5 分钟游戏内时间，具体倍率在 `idle-config.ts` 调整）：

```typescript
const IDLE_INTERVAL_MS = 10_000;

useEffect(() => {
  const timer = setInterval(() => {
    const state = getState();
    if (state.meta.prologue_complete) {   // 开场完成后才开始积累
      const rates = calcIdleRates(state);
      applyIdleAccumulation(rates, IDLE_INTERVAL_MS / 60_000);
    }
  }, IDLE_INTERVAL_MS);
  return () => clearInterval(timer);
}, []);
```

`applyIdleAccumulation` 直接调用 `setState()` 更新资源，**不经过 LLM，不生成叙事**。由于 `setState` 会通知所有 `subscribe` 监听器，资源面板数字会自动刷新，无需额外处理。

### 4.5 离线补算实现

**触发时机**：游戏加载时（`CourtPage` `useEffect` 挂载阶段），在 `initGame()` 之前执行。

**流程**：

```
1. 读取存档中的 meta.last_idle_tick_at
2. 计算 offlineMs = Date.now() - new Date(last_idle_tick_at).getTime()
3. 上限截断：offlineMs = Math.min(offlineMs, 24 * 60 * 60 * 1000)
4. 换算为分钟数：offlineMins = offlineMs / 60_000
5. 若 offlineMins > 1，执行一次性补算：
   applyIdleAccumulation(calcIdleRates(state), offlineMins)
6. 更新 meta.last_idle_tick_at = new Date().toISOString()
7. 弹出离线收益通知（见 4.6）
```

`last_idle_tick_at` 在每次实时 `setInterval` 执行后也需更新，防止下次补算重复计算。

### 4.6 离线通知文案

不走 LLM，模板字符串直接写死，由 Mimo 实现：

```
离开期间（{hours}小时{minutes}分钟），帝国持续运转——
粮仓：+{food_gained} 石　国库：+{fiscal_gained} 两
军费消耗：{military_drained} 两
```

显示方式：复用现有的 `Toast` 组件或在叙事框顶部插入一条灰色提示条，3 秒后自动消失。

### 4.7 资源显示变更

放置系统启用后的 UI 调整（最小改动原则）：

1. **数字刷新**：由 `subscribe` 自动驱动，`setState` 触发时组件重渲染，无需额外逻辑。
2. **速率标注**（可选，优先级低）：资源数字旁括号内显示 `+x/min`，让玩家感知速率。作为后续迭代，不阻塞当前实现。

## 五、GameState 变更需求

### `types.ts` 的 `Meta` 接口新增字段

```typescript
interface Meta {
  // 现有字段不改动
  // version, created_at, last_saved_at, save_slot, game_year, real_time_played_ms

  // 新增：开场阶段追踪
  prologue_phase: 'awakening' | 'guoshi_intro' | 'complete';
  prologue_complete: boolean;

  // 新增：放置系统时间戳
  last_idle_tick_at: string;   // ISO8601，上次放置积累的时间点
}
```

**`createEmptyGameState()` 中的初始值**：
```typescript
meta: {
  // ... 现有字段 ...
  prologue_phase: 'awakening',
  prologue_complete: false,
  last_idle_tick_at: new Date().toISOString(),
}
```

**旧存档兼容（`save.ts` 加载时 patch）**：
```typescript
if (!state.meta.prologue_phase) {
  state.meta.prologue_phase = 'complete';      // 旧档直接跳过开场
  state.meta.prologue_complete = true;
}
if (!state.meta.last_idle_tick_at) {
  state.meta.last_idle_tick_at = state.meta.last_saved_at;
}
```

- [ ] 执行面板包含：临朝听政、前往（含地点子菜单）、召见、查阅史册、颁布政策
- [ ] 选择"临朝听政"后进入原有 gameTick 流程，游戏正常运行
- [ ] 读取旧存档时，开场不重新触发
- [ ] 整个开场过程中，不出现"Tick"、"政策面板"、"点击"、"按钮"等现代 UI 词汇

### 放置系统验收

- [ ] 开场完成后空置 30 秒，`food` 和 `fiscal` 数值有可见增加
- [ ] `military` 在军费为正时有缓慢下降

- [ ] 离线超 24 小时时，收益按 24 小时上限计算
