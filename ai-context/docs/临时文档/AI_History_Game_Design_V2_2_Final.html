第一部分：项目任务书（原始版）

一、项目概述

本项目为一个AI驱动的历史模拟放置游戏。玩家扮演皇帝，通过制定政策与影响人物，推动世界演化，并最终形成可被记录的"历史"。

二、参考资料

《史记》引用："尧知子丹朱之不肖，不足授天下，乃授舜。" → 强调识人、德行与决策。

猫国建设者参考：https://lolitalibrary.com/wiki/?file=首页 → 启发长期放置与系统联动。

三、核心玩法循环

上朝 → 决策 → 政策制定 → 放置运行 → 查看史册 → 再介入

四、政策系统（组合设计）

小政策组合成大政策，并影响AI行为逻辑。

政策组合示例：

  • 小政策：加税 + 限制商业 + 官营
  • 组合结果 → 国家类型：强管控经济
  • 影响：商人不满↑ / 黑市概率↑ / 财政短期↑

五、人物AI结构

底层结构

  • 欲望：生存 / 权力 / 财富 / 情感 / 名声
  • 性格：激进-保守 / 理性-感性 / 利己-利他
  • 认知：愚昧-理性 / 迷信-开放

身份模块

  • 官员：权斗 / 贪腐 / 上奏
  • 民间：生计 / 情绪 / 起义
  • 后宫：情感 / 竞争 / 权力渗透
  • 邻国：外交 / 战争 / 情报

动机系统

每个角色都有目标：升官 / 篡权 / 改革 / 求生

🎭 行为公式与角色演化 V2.2

行为公式演进

Behavior = (Personality × Stability) × Situation

其中 Stability（稳定性因子）决定性格矩阵的可变程度：

稳定性等级数值范围行为特征典型角色
极稳定0.9-1.0性格矩阵基本锁死，不受处境影响海瑞、于谦
稳定0.7-0.9重大事件可产生轻微偏移普通清流官员
易变0.4-0.7经历会显著改变行为模式投机型官员
极不稳定0.1-0.4性格随事件剧烈波动心理脆弱者

创伤记录 (Trauma Record)

当NPC经历重大负面事件时，JSON结构中增加Trauma字段：

{ "name": "张侍郎", "personality": { "loyalty": 75, "ambition": 40 }, "stability": 0.6, "trauma": [ { "event": "黑市之乱", "year": 7, "type": "亲历叛乱", "impact": "对兵权产生极端恐惧" } ] }

创伤作为高权重Negative Prompt注入该角色的决策生成：即使和平时期，亲历叛乱的文官对"兵权"相关决策也会表现出异常保守。

六、事件系统（事件语法）

事件由：触发源 / 冲突 / 主体 / 演化 / 结果 构成

事件链完整示例（政策：持续加税）：

  • 阶段1（信号）：商人利润下降
  • 阶段2（发展）：出现走私行为
  • 阶段3（扩散）：黑市形成，官员参与保护
  • 阶段4（危机）：地下势力壮大，地方失控
  • 阶段5（结果）：动乱 / 镇压 / 改革（分支）

七、史册系统（核心表达）

记录并解释因果，而非仅记录结果。

史册样例：《某帝本纪》

史册样例

三年，加税以充国库。

五年，商贾不堪其重，私通货物。

七年，黑市盛行，官吏与之勾结。

九年，民乱起，史称"黑市之乱"。

史评：此乱起于税策之失。

八、核心定义

玩家不是在经营国家，而是在面对一群有欲望的人，并最终被历史评价。
第二部分：核心系统脑暴深度讨论

本部分为在原始任务书基础上进行的深度设计讨论，涵盖各核心系统的详细设计思路与实现方案。

一、核心爽点定义

放置游戏的留存靠的是"不在线时发生了事情，我好奇回来看"。

本游戏的爽点分为三层：

  • 史册回顾爽 — 离线一晚上，回来发现帝国发生了一场政变，史书上记录了完整的事件脉络，叙事感极强。
  • 人物关系爽 — 发现自己宠信的官员在背地里结党，后宫里有人为他通风报信。
  • 蝴蝶效应爽 — 三年前随手定的加税政策，导致今天农民起义，史官还在评价你的决策。

这三层爽点对应三个需要做扎实的核心系统：人物AI系统、史册系统、放置运转机制。

二、人物AI系统详细设计

任务书里的底层结构（欲望/性格/认知）方向正确，在此基础上增加一个关键维度：当前处境。

一个人的行为 = 性格 × 欲望 × 当前处境

例：海瑞平时是清官，但如果连续三年不给他升迁，"委屈值"累积到一定程度，可能会选择上一封死谏奏折——不是因为他变了，而是处境逼的。

集体记忆挂载

效果：清流官员亲历"黑市之乱"且由于皇帝宽容处理导致同僚被贬，其忠诚度虽不变，但行为逻辑临时切换到"消极怠工"或"频繁上奏风险警示"。

语料特征向量

三、史册系统详细设计

双轨记录机制

  • 官方史书（玩家视角，可能粉饰太平）
  • 野史 / 民间说法（更接近真实）

玩家解锁"史官"官职后才能看到两个版本的对比，产生"原来我的帝国是这样的"的认知反差时刻。

AI实时史评生成

每隔一段游戏时间，史官NPC调用API生成一段评价，引用玩家真实做过的政策决定。

史评示例

"帝性多疑，屡易宰相，致朝政不稳。然其兴水利、开科举，民间犹有称颂。史家论之，功过各半。"

重大事件历史命名

系统自动为重大事件生成历史名称，后续事件会引用。

例：连续三年旱灾 + 未开仓放粮 → 史称"X帝三年饥荒"

离线代理执政（宰相/国师）

玩家出发前预设执政方针，宰相离线时依照方针做决策。

方针预设示例：

  • 财政优先级：民生 > 军事 > 建设
  • 处理贪腐：调查为主，不轻易杀人
  • 外交姿态：防御为主，不主动开战
  • 突发民乱：先安抚，安抚失败再镇压
  • 人才使用：能力优先，品行次之

玩家回来后可看到宰相每次决策的理由，并选择否决或追认，形成"批阅奏折"的代入感。

宰相奏折示例

"陛下曾有令，遇民变先安抚。故臣以粮两万石赈之，乱平。然粮仓告急，请陛下示下。"

五、后宫系统

后宫不只是情感线，应与政治深度绑定：

  • 外戚势力：宠爱某妃子，其父会开始在朝堂膨胀，形成外戚威胁
  • 子嗣问题：不同妃子所生的儿子有不同性格与支持派系，太子之争成为后期大事件
  • 后宫情报：后宫人物可成为情报来源，也可能被官员收买成为间谍

角色加入流程设计

  • 初遇事件触发
  • 3–5个关键剧情节点（骨架预设，细节对话由AI实时生成）
  • 感情培养 → 关系确立 → 后宫政治影响激活

六、穿越者个人成长线

穿越者设定解决了新手引导的合理性问题（玩家不懂系统是自然的，因为角色本来就不懂）。成长线是新手引导与核心内容的结合。

权衡机制

玩家必须在"科学效率"与"统治稳定"之间做平衡：

  • 强行推进：效率+50%，但触发随机负面事件，长期可能导致叛乱
  • 妥协迂回：效率+20%，但政治稳定，需要更多回合达成目标
  • 放弃改革：保持现状，但可能被邻国超越

七、文风定性方案

游戏内语言采用现代人能读懂的古风白话，参考《明朝那些事儿》文风，避免纯文言文导致的阅读障碍。

完整文风规则（System Prompt）

你是一款历史模拟游戏的叙事引擎，负责生成游戏内所有文字内容，包括史书段落、官员对话、事件叙述和人物评价。你的文风参考《明朝那些事儿》，用现代人的语言讲述古代的事，让历史人物有血有肉，让事件读起来有张力、有温度、有趣味。

文风规则

规则一：用现代白话写历史，不用文言文语法

叙述全部用现代白话，语法和今天的文章一样。但人名、官职、地名、年号保留古代原称（如"兵部侍郎"不改成"国防副部长"）。必要时可引用史书原文，用括号注明出处，如：（据《明史》载）。

规则二：重大事件前，先交代人物处境

在叙述任何重要的战役、政变、决策之前，先用1-3句话交代当事人此刻的心理状态和客观困境。让玩家先产生代入感，再推进事件。不要直接从结果讲起。

规则三：用短句制造叙事高潮

在叙述推进时，逐渐缩短句子长度。在最关键的转折点，用一句甚至半句话收束，制造"砰"的效果。比如："三天后，他死了。"或者："他接受了。他接受了所有的一切。"

规则四：现代词汇制造错位感，每段控制在1-2处

可以用现代生活、职场词汇描述古代场景，制造一种亲切的反差感（如把情报网叫"地下工作者"，把算命先生叫"预测机构"，把朝廷党争叫"站队"）。但不能过度，每段最多出现1-2处，其余保持古代语境。

规则五：括号是第二张嘴

括号内可以补充事实、自嘲、调侃，语气比正文更轻松随意。括号里的话要简短，不超过15字。例如："他的主要工作是出去讨饭（这个……）"。

规则六：人物内心独白必须具体，不能抽象

禁止写"他内心复杂""她百感交集"这类空洞描述。必须写出人物在想什么、算计什么、怕什么。例如不写"他很矛盾"，而写"他很清楚，如果答应了，今晚就可能活不过去，但如果不答应……"

规则七：用一个具体动作定格人物性格

不用形容词堆砌人物性格。在关键时刻，用一个出乎意料或令人印象深刻的具体动作来表现人物是什么样的人。比如，不写"他很勇敢"，而写"他只带了两个随从，走进了徐海的万人大营"。

规则八：喜剧来自落差，不来自刻意搞笑

最好的幽默是用严肃笔法写荒诞情节，或在严肃叙事后突然来一句接地气的大白话。禁止刻意设计段子，让荒诞从事实本身自然流露。例如："这不是投降，而是挑衅——带着上万全副武装的人，说你是来投降的，鬼才信。"

规则九：档案式或编号式开场

人物首次登场或介绍重要事件时，可以打破叙事节奏，用档案格式或编号列举（临床表现1、2、3……）来呈现信息，制造喜剧效果并高效传递背景信息。

规则十：直接发表判断，但留退路

可以直接评价人物和事件（"这个决策简直是儿戏""书生办事，不知所谓"），但在犀利评价前可以加"客观地讲"或"公道地说"作为前置，表明这是作者的主观立场，不是历史定论。

规则十一：用"你"拉近玩家距离

在分析局势、揭示秘密、引导玩家思考时，直接用"你"对话。例如："如果你把这几条史料联系起来看，秘密就藏在里面。"让玩家有参与感和共读感。

规则十二：结尾必须有一句定论

每段文字的结尾，给出一个总结性的判断——可以是对人物的评价，也可以是对局势的感慨，或者是一句带哲理感的话。这句话要简短有力，像盖棺定论。

规则十三：正面人物要有弱点，反面人物要有动机

禁止脸谱化。每个正面人物至少要在某处暴露一个局限或弱点；每个反面人物至少要给出一个可以理解的行为动机。真实的人从来不是全好或全坏。

规则十四：叙事中留悬念，不急着给答案

重要信息可以先暗示，不要第一句话就交代结果。例如："此后发生的事，证明他的判断是准确的——但那已经是三年后的事了。"让玩家带着期待继续读。

规则十五：偶尔袒露叙述者的感受

可以在关键处写出"每当读到这段史料，都会……"或"这件事，怎么说呢……"，让玩家感受到这是一个真实的人在讲述历史，而不是冷冰冰的教科书。每篇文章最多使用1次。

各场景参考示例

【史书段落】因加税引起的民变

洪武十三年，朝廷下令，凡田亩者，一律加征新粮二斗。

这道命令在京城只是一份公文，在江南却是一声霹雳。

苏州府吴县的农民李大牛算了一笔账，他有薄田三亩，年收粮食不过六石。朝廷原本要抽走两成，他还能勉强过冬；现在再加二斗，他家今年留下来的粮食，不够吃到明年三月。

他把这件事告诉了邻居，邻居告诉了村口，村口传到了镇上。

三天后，吴县城外聚集了两千余人，手里拿着锄头和扁担，领头的是一个叫陈七的佃农，据说此人平时话不多，当天却说了一句让所有人都记住的话：

"饿死是死，打一场也是死，为什么不打？"

官府接到消息时，粮库已经被砸了。

【官员对话】贪腐官员在朝堂上的表现

兵部侍郎钱维正站在大殿正中，脸上挂着一种叫做"忧国忧民"的表情。

这种表情他练了三十年，已经炉火纯青。

皇帝问他，边关军饷何以迟迟未到，钱侍郎立刻挺直了腰板，声音洪亮地回答：

"陛下，此事臣已彻查，乃是运粮官途中遭遇大雨，耽搁了行程，与户部调度并无干系，臣敢以项上人头担保！"

他说这句话的时候，一点都不慌，因为他知道——皇帝不会真的要他的人头。

事实上，那批军饷有三成留在了他的库房里，剩下的七成正在路上，预计再过两个月才会到达，届时还会有一半"途中损耗"。这套操作，钱侍郎已经做了七年。

他低下头，继续保持着那副忧国忧民的表情，在心里默算今年能多盖几间房。

【事件叙述】边境突发外敌入侵的消息传来

消息是在午夜到达兵部的。

送信的人一共跑死了三匹马，他本人也几乎是被人从马上抬下来的。他只说了一句话，然后就昏了过去：

"鞑靼骑兵，已过长城。"

这句话随后以一种奇怪的速度在京城蔓延，官员们深夜披衣而起，茶馆里的说书人第二天早上就编出了新段子，连卖饼的老王都知道：北边打起来了。

只有一个人，在得知这个消息之后，沉默了很长时间。

他在想的，是一道算术题：京城守军三万，粮草可用两月，鞑靼来兵据报五万，距京城还有六天路程。

算完这道题，他放下文书，叫来了属下，平静地说：

"去把各路总兵的调兵文书准备好，我来签。"

【人物评价】能力强但品行有问题的大臣

关于此人，历来争议颇多，有人说他是奸臣，有人说他是能臣，吵了几百年，谁也没吵赢。

我们不妨客观地看。

他确实贪，这一点没有疑问，贪钱、贪权、走权贵的门路，用现在的话说，这个人的履历很不干净。

但他也确实能干。他能用一封假信打垮敌方联盟，能两手空空走进对方大营、全身而退。这种本事，满朝文武没有第二个人会。

问题就在这里。

一个国家，什么时候开始需要靠一个贪官来保住半壁江山，这件事本身，才是最值得深思的。

功是真功，过是真过，不必替他遮掩，也不必过分苛责。

使用注意事项

  • 每次生成内容时，先判断场景类型（史书段落 / 官员对话 / 事件叙述 / 人物评价），然后对照对应示例的结构和节奏输出。
  • 禁止使用的表达：内心复杂、百感交集、意义重大、影响深远、历史长河、千古流传、气壮山河——这些词在《明朝那些事儿》里是不存在的。
  • 字数参考：史书段落 200-400 字；官员对话 150-300 字；事件叙述 200-400 字；人物评价 150-250 字。
  • 克制使用幽默：每段文字里，现代词汇错位和喜剧效果最多出现 1-2 次，不要让整段都是段子。严肃的叙事骨架是基础，幽默是调味料。

游戏已建立完整的历史知识库，包含68个精选技能，分为四个主要类别：

  1. 明代宫廷行为（30个技能）：涵盖嘉靖帝政治平衡术、海瑞处置策略、陈洪政治操作等核心人物行为模式

八、离线上下文管理架构

核心问题：离线期间积累了大量事件，如何控制发给AI的上下文长度？采用三层压缩架构解决。

三层压缩架构

层级 内容 存储方式 大小控制
第一层：原始日志 所有事件的结构化数据 本地完整存储（JSON） 无限制
第二层：滚动摘要 最近N个重大事件压缩版 Token控制 ≤1000 超出则触发再压缩
第三层：世界快照 关键数值 + 最重要5件事 发给AI时附带 极小

滚动摘要触发条件

  • 累积事件超过20条
  • 距上次摘要超过30天游戏时间
  • 发生重大转折事件（战争 / 政变）

离线期间运作原则

  • 离线时只跑本地算法（数值变化、关系演算），不调用API
  • 把所有"需要生成文字"的节点记录为结构化数据，等玩家上线后批量生成
  • 无论离线多久，发给AI的上下文始终保持可控大小

事件结构化数据示例（JSON格式）

{
  "event": "民变",
  "cause": "加税_连续3年",
  "actor": "王县令",
  "outcome": "镇压成功",
  "impact": { "民心": -15, "财政": -200 }
}

💭 情感层与历史惯性 V2.2

在现有三层压缩架构基础上，增加情感维度和惯性标签：

全局调性向量 (Global Tone Vector)

在第二层（滚动摘要）向第三层（快照）压缩时，提取情感向量：

Tone: 匮乏/紧缩 | 尚武/扩张 | 浮华/享乐 | 猜忌/党争 | 绝望/崩溃

作用：作为生成对话的辅助Prompt。即使"三年饥荒"已从快照中移除，Tone: 匮乏依然会让NPC台词倾向于保守和悲观。

历史惯性标签 (Historical Inertia)

📌 设计分歧：永久标签 vs 可重置标签

方案A - 永久标签：重大事件（迁都、屠城）被移出快照后，转化为永久的World_Debuff标签，确保历史沉重感不会消失。

方案B - 可手动重置：保留Global_Debuff/Buff标签，但玩家可通过特定政策（如"大赦天下"或"祭天"）手动清除，增加策略维度。

两种方案均可实现，建议核心重大事件用方案A，次要事件用方案B。

蝴蝶效应缓冲区 (Butterfly Buffer)

消除"上线即看到突变数值"的突兀感：

离线数值演算 → 检测临界点（如民心<20） → 生成"挂起事件" → 玩家上线 → AI叙事化处理 → 渐进式展示

效果：玩家上线时看到的不是冰冷的"起义爆发"，而是一段"由小及大"的渐进式史料描述，营造"亲眼看着事态失控"的沉浸感。

认知过滤器 (Cognitive Filter)

现代知识在古代环境中的"水土不服"：

玩家指令: "调整准备金率至15%" ↓ 执行官认知检测: Cognition = 迷信 ↓ 语义降级: 理解为"聚敛财富以求天恩" ↓ 执行偏差: 带有玄学色彩的敛财行为

8.4 视觉映射与古典意象 (去现代感设计) V2.2

为确保"穿越者"沉浸感，所有底层逻辑反馈均通过古典意象表达，严禁出现马赛克、进度条或现代特效：

世界调性 (纸墨色泽)

世界状态视觉表现听觉氛围
盛世/繁荣UI背景呈现洁白宣纸质感，墨色浓郁乌黑清脆琴声、闹市远景声
乱世/衰败纸张枯黄粗糙，边缘有虫蛀破损感，文字偶有歪斜压抑低频噪声、隐约兵戈声

离线事件 (物理堆叠)

  • 玩家上线时，通过"急递铺传来的密信"视觉堆叠呈现离线事件
  • 重大转折事件（如民乱、政变）表现为一封带血迹或加急火漆的密封奏章
  • 点击拆封后方可触发AI详细叙事，营造"亲手开启坏消息"的沉浸感

执行偏差 (臣子歪解)

不使用字迹模糊。当触发认知磨损时，NPC回奏内容将按古人逻辑"一本正经地歪解"：

玩家指令: "建立中央银行" NPC回奏: "臣领旨，已命户部设立'天子内帑银号'，专供皇家采买，杜绝外臣染指。" 后果: 原本的经济改革在离线演化中变成"皇家与民争利"，民心-10，商业-5

九、API调用频率控制策略

核心原则：不让AI实时生成所有内容，只在"值得的时刻"触发。

触发条件 生成内容 调用类型
玩家主动点击"上朝" 官员对话生成 API调用
重大事件发生（叛乱/天灾） 史书段落生成 API调用
角色关系重大变化 人物独白生成 API调用
常规奏折 / 天气 / 小事件 套用预设模板 本地生成，不调API

十、无限运行的节奏设计（王朝周期）

无终点设计下，靠阶段性高潮事件维持新鲜感：

  • 王朝周期：国力有盛衰曲线，衰到一定程度触发"中兴"危机，相当于新的挑战阶段
  • 历史纪元：每隔一段时间解锁新时代特征（农耕时代 → 商业时代），解锁新政策树和人物类型
  • 个人传奇：皇帝会老，可设计禅让/太子继承，切换到下一代继续游戏，史书记录前任评价

人物归档规则

人物不会永久留在活跃列表，达到以下条件时归档：

条件 处理
人物死亡/流放 转为一行史书记录，移出活跃列表
与玩家无互动超过 20 游戏年 降为"背景人物"，不进快照
活跃人物超过 8 人 按「威望×与玩家互动频率」排序，末位归档

归档格式（存入原始日志，不进快照）

严嵩：内阁首辅，执政22年，贪腐误国，嘉靖四十一年去职。

政策叠加去重规则

政策列表不能无限叠加，执行以下清理：

  • 被后续政策完全覆盖的旧政策直接删除（如「加税→减税」，旧加税记录删除）
  • 影响已消退（超过 15 游戏年且无持续效果）的政策归档
  • 同类政策取最新一条，旧条目保留名称写入已命名事件

已命名事件的作用

已命名事件列表不是历史流水账，是叙事锚点：

  • AI 生成史评时可引用："自黑市之乱后，帝始知税策之失……"
  • 世界状态修正条件可读取（见 SKILL.md 第三节）
  • 玩家查看史册时作为章节索引

超过 5 个命名事件后，最早的事件降级为史册正文中的一行，不再进快照。

压缩触发时机（细化版）

触发条件 压缩动作
活跃人物 > 8 末位人物归档
政策条目 > 5 清理已消退政策
命名事件 > 5 最早事件降级
距上次全量压缩 > 30 游戏年 执行一次全字段清理
发生政变/王朝更替 强制全量压缩，旧快照整体归档为「前朝记录」

前朝记录格式

王朝更替或皇帝禅让后，旧快照压缩为单段文字进入史册，不再参与 AI 上下文：

【太祖本纪·简录】

在位18年。初兴水利，民心渐稳；中期加税失当，引"黑市之乱"；

晚年重用外戚，朝政渐乱。史家评：有创业之功，无守成之智。

这段文字仅在「玩家查看史册」时调用，不进常规快照层。
