# 02 — V3 文档问题清单

评审对象：`../AI_History_Game_Design_V3_Executable.html`。

**评审结论**：V3 是优秀的**方向性**文档，但不是**工程可落地**文档。问题按 P0/P1/P2 分级列出；每条标注处置去向。

---

## P0 — 不解决就动不了工

### P0-1. 运行环境缺失
- **问题**：全文没有指出游戏跑在浏览器、Node、还是 Electron；LLM key 放哪；是否需要后端。
- **处置**：已在 `01_architecture.md §1` 明确——GitHub Pages 纯静态 + 玩家自填 key + 无后端。

### P0-2. 没有 GameState 单一 schema
- **问题**：V3 定义了 6 种局部 JSON（decision_trace、event、conflict、snapshot、memory、代理执政），但没有顶层结构把它们串起来。没有它就存不了档、不能回放、UI 无法渲染、AI 读写不一致。
- **处置**：已在 `04_gamestate_schema.md` 完整起草。

### P0-3. API 成本没算过
- **问题**：默认管线是"决策 JSON → 叙事改写"两次调用，多 Agent 冲突还要 N+1 次。以 4 官员计，一次上朝 5 次 API。缺预算、缺降级策略、缺缓存。
- **处置**：已在 `01_architecture.md §3` 用"三档生成 + 延迟生成 + 语义缓存"方案处理；多 Agent 用"并发解读 + 单次仲裁"合并为 N+1 次但墙钟时间 ≈ 2 次，见 `03_fixes_p1.md §2`。

### P0-4. 文风方案自相矛盾 ✅ 已解决
- **问题**：V3 §7 宣称删除 70% 硬规则改风格标签；但独立文档有完整 15 条。AI 不知道听谁的。
- **处置**：已拍板以 15 条为唯一权威；V3 §7 做最小修订，只保留风格标签作为"切换开关"，指向独立文档。详见 `08_decisions.md` 决策 3。

## P1 — 会影响体验但可以滚动修（本方案全部填充）

### P1-5. Layer 1 "世界规则层"只有目录没内容
- **问题**：V3 §2.1 Layer 1 只写了"行为约束 / 禁止规则 / 全局机制"三行 bullet，无具体条目。无内容 = 无约束。
- **处置**：已在 `03_fixes_p1.md §1` 起草 20 条具体规则，分物理时序 / 叙事约束 / 机制底线三类。

### P1-6. 多 Agent 冲突没有调度器
- **问题**：谁先发言？并发还是串行？权力计算的具体公式是什么？V3 §2.4 只给 JSON 壳。
- **处置**：已在 `03_fixes_p1.md §2` 定义"并发解读 + 单次仲裁"调度策略 + 权力公式草案。

### P1-7. 人物记忆无界增长
- **问题**：`trauma[]` / `key_events[]` 一直累积。几十年后单个 NPC 上下文就爆。V3 无淘汰机制。
- **处置**：已在 `03_fixes_p1.md §3` 定义 LRU（容量上限 + 情感强度 × 时间衰减公式）。

### P1-8. 离线演算和在线叙事的接缝没定义
- **问题**："本地算法跑数值"——算什么？公式在哪？"挂起事件"数据结构？
- **处置**：已在 `03_fixes_p1.md §4` 给出 5 个核心数值的 tick 公式。

### P1-9. 图位完全没规划
- **问题**：V3 §10.3 只说宣纸/枯黄视觉，但 NPC 头像、事件插画、地图、icon 全缺席 schema。
- **处置**：已在 `01_architecture.md §5` 和 `04_gamestate_schema.md` 的 `Visual` 字段处理。

### P1-10. skill 知识库没和 V3 对接
- **问题**：`emperor-game-knowledge-base` 有 46+ 子技能，`skill_routing_supplement.md` 有完整路由字典，但 V3 Prompt 模板里完全没提"技能注入"这一层。两套系统还是同一套？文档没合并。
- **处置**：已在 `03_fixes_p1.md §6` 和 `05_prompt_layers.md` 把路由字典正式吸收进 Layer 3 拼装流程。

## P2 — 锦上添花

### P2-11. 中英文 key 混用
- **问题**：`loyalty` / `当前朝代` 混用。AI 处理时容易错乱。
- **处置**：已在 `03_fixes_p1.md §7` 统一——代码层英文 key，UI 层中文 label 映射。

### P2-12. 内禅密旨作为下一代 System Prompt 需要版本管理
- **问题**：V3 §10.5 的设定很好，但每代都叠加 prompt 会越来越长。
- **处置**：已在 `03_fixes_p1.md §8` 定义 diff 链 + 5 代折叠压缩策略。

### P2-13. 风格标签覆盖不全
- **问题**：V3 §7 只给 3 个（史官视角/冷幽默/克制叙述），覆盖不够。
- **处置**：本次不展开。文风详情由 15 条独立文档管，标签只做切换开关。后续若需要新标签，在 `05_prompt_layers.md` 追加。
