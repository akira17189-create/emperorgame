# 靖朝政策系统设计文档

**设计理念**：政策是皇帝治理国家的核心手段，每条政策不仅影响资源数值，更会改变NPC立场、王朝走向，并触发朝堂冲突与民间反应。

**政策分类**：内政（6个）、军事（5个）、仙道（6个）、工业（5个），共计22个可玩政策。

---

## 一、内政政策（6个）

### 1. 均田令
**朝堂描述**："朕闻民为邦本，本固邦宁。今下诏推行均田令，重新丈量田亩，限制豪强兼并，使耕者有其田，饥者有食粮。"

**效果**：
- **Core层**: food +15%, population +5%, morale +10
- **Support层**: tax_rate +3, land_fertility +8
- **Flavor层**: 记录为"仁政"，增加皇帝声望

**冲突触发条件**：
- 大地主NPC（财富>70）忠诚度下降20
- 清流派支持度+15，帝党派支持度-10
- 可能触发"豪强抵制"事件

**负面后果**：若执行不力（agriculture40），可能导致"土地丈量舞弊"，food反而-10%

### 2. 科举改革
**朝堂描述**："八股取士，拘泥死板，难选真才。朕欲改革科举，增设算学、实务二科，不拘一格降人才。"

**效果**：
- **Core层**: fiscal +5%, morale +8
- **Support层**: commerce +10, eunuch -5（削弱宦官影响力）
- **Flavor层**: 新增"改革派"官员派系

**冲突触发条件**：
- 翰林院NPC（knowledge>80）激烈反对
- 宦官派支持度-15，军队派观望
- 可能触发"科举舞弊案"

**负面后果**：若education<50，改革可能失败，导致"士林哗变"，morale -15

### 3. 盐铁专卖
**朝堂描述**："盐铁乃国之命脉，岂容私商操控？自即日起，盐铁收归官营，统一调配，以充国库。"

**效果**：
- **Core层**: fiscal +25%, threat +5（可能激起民怨）
- **Support层**: commerce -15, eunuch +10（宦官借机扩权）
- **Flavor层**: 记录为"聚敛之政"

**冲突触发条件**：
- 盐商NPC（wealth>60）忠诚度归零
- 帝党派强力支持+20，清流派激烈反对-25
- 可能触发"盐商暴动"

**负面后果**：若corruption>30，专卖权被宦官把持，fiscal实际增益减半

### 4. 水利兴修
**朝堂描述**："水能载舟，亦能覆舟。朕拨内帑三十万两，于黄河、淮河沿岸兴修水利，防涝抗旱，保百姓安宁。"

**效果**：
- **Core层**: food +20%, population +8%
- **Support层**: disaster_relief +15, military_cost +5（占用军费）
- **Flavor层**: 记录为"德政"，增加皇帝仁德值

**冲突触发条件**：
- 工部官员（engineering40）可能贪腐
- 军队派反对-10（占用军费）
- 可能触发"河工贪墨案"

**负面后果**：若30，工程质量差，可能次年爆发水灾

### 5. 裁撤冗员
**朝堂描述**："朝廷冗员充斥，人浮于事。朕决心裁撤三成闲散官员，精简机构，提高行政效率。"

**效果**：
- **Core层**: fiscal +15%, morale -10（官员恐慌）
- **Support层**: eunuch -8, faction -5（削弱派系）
- **Flavor层**: 记录为"厉行节俭"

**冲突触发条件**：
- 所有官员NPC忠诚度-15
- 宦官派激烈反对-20
- 可能触发"官员集体请辞"

**负面后果**：若loyalty_to_em40，可能引发政变

### 6. 开放海禁
**朝堂描述**："闭关锁国，无异坐井观天。朕决定有限开放海禁，设市舶司，与西洋诸国通商，以开财源。"

**效果**：
- **Core层**: fiscal +30%, threat +8（倭寇风险）
- **Support层**: commerce +25, military +10（需要加强海防）
- **Flavor层**: 引入"西洋技术"概念

**冲突触发条件**：
- 保守派NPC激烈反对
- 清流派分裂，部分支持改革
- 可能触发"倭寇大规模入侵"

**负面后果**：若military<60，海防薄弱，commerce增益减半

## 二、军事政策（5个）

### 7. 边军改制
**朝堂描述**："边军疲敝，军纪涣散。朕欲推行'靖边军法'，裁撤老弱，招募精壮，重振边关军威。"

**效果**：
- **Core层**: military +25%, fiscal -15%（军费大增）
- **Support层**: threat -10, morale +5（民心振奋）
- **Flavor层**: 新增"戚家军"部队

**冲突触发条件**：
- 被裁撤将领可能叛乱
- 军队派强力支持+25
- 可能触发"边军哗变"

**负面后果**：若fiscal<30，军费不足，改革失败

### 8. 火器营组建
**朝堂描述**："西洋火器，威力惊人。朕命工部仿制佛郎机炮，组建专业火器营，以应边患。"

**效果**：
- **Core层**: military +30%, industry +10
- **Support层**: technology +15, fiscal -20
- **Flavor层**: 开启"军事科技"树

**冲突触发条件**：
- 保守武将反对"奇技淫巧"
- 工部官员（engineering>60）积极支持
- 可能触发"火器炸膛事故"

**负面后果**：若technology<40，仿制失败，浪费军费

### 9. 军屯制恢复
**朝堂描述**："养兵千日，用兵一时。朕恢复前朝军屯制，令边军战时为兵，闲时为民，自给自足。"

**效果**：
- **Core层**: fiscal +10%（节省军费）, food +15%
- **Support层**: military_cost -20, land_fertility +5
- **Flavor层**: 记录为"养兵于民"

**冲突触发条件**：
- 军队派反对-15（降低待遇）
- 文官集团支持+10（节省开支）
- 可能触发"军屯侵占民田"

**负面后果**：若dis50，军屯效率低下

### 10. 水师扩建
**朝堂描述**："倭寇肆虐，海疆不宁。朕拨专款扩建水师，造福船百艘，巡弋东海，保境安民。"

**效果**：
- **Core层**: threat -15, fiscal -25%
- **Support层**: naval_power +30, commerce +10（保护商路）
- **Flavor层**: 新增"郑和船队"历史典故

**冲突触发条件**：
- 内陆武将反对"重海轻陆"
- 沿海官员强力支持
- 可能触发"造船贪腐案"

**负面后果**：若corruption>40，船质量差，naval_power增益减半

### 11. 边防城墙加固
**朝堂描述**："北虏南倭，边关永无宁日。朕征民夫十万，加固九边城墙，增设烽火台，筑钢铁防线。"

**效果**：
- **Core层**: threat -20, population -5%（征用民夫）
- **Support层**: defense +40, morale +5
- **Flavor层**: 记录为"固若金汤"

**冲突触发条件**：
- 民夫NPC满意度大降
- 军队派强力支持+20
- 可能触发"民夫暴动"

**负面后果**：若morale<30，工程进度缓慢

## 三、仙道政策（6个）

- 可能触发"丹药中毒"

**负面后果**：若tao30，炼丹失败，emperor.health反而-10

### 13. 观星台修建
**朝堂描述**："天象示警，不可不察。朕拨内帑修建观星台，观测天象，预知吉凶，以安天下。"

- **Core层**: threat -5（预知灾害）, fiscal -15%
- **Support层**: astronomy +30, disaster_relief +10

- 钦天监官员积极支持
- 可能触发"天象误读"

### 14. 灵气勘探

### 15. 道观免税

**负面后果**：若taoist>70，道教干政，eunuch +10

### 16. 仙草种植

### 17. 龙脉祭祀

## 五、政策数据结构

```typescript
export interface Policy {
  id: string;
  name: string;
  category: 'internal' | 'military' | 'taoist' | 'industrial';
  courtDescription: string;  // 朝堂颁布时的文案
  effects: PolicyEffect[];
  conflictTriggers: ConflictTrigger[];
  negativeConsequences: NegativeConsequence[];
  duration: number;  // 政策持续时间（年）
  isActive: boolean;
  enactedYear: number;
}

export interface PolicyEffect {
  target: string;  // 影响的字段，如"resources.food"
  type: 'add' | 'multiply' | 'set';
  value: number;
  layer: 'core' | 'support' | 'flavor';
}

export interface ConflictTrigger {
  npcCondition: string;  // NPC触发条件，如"wealth>70"
  factionChange: Record<string, number>;  // 派系支持度变化
  eventRisk: string;  // 可能触发的事件ID
}
```

## 六、政策系统与现有引擎衔接

1. **政策颁布**：玩家在CourtPage选择政策 → Policy Interpreter解析 → 生成Policy对象
2. **即时效果**：政策生效，立即影响资源，触发NPC反应
3. **Tick计算**：每个Tick计算政策持续效果
4. **冲突仲裁**：政策触发冲突时，调用arbitration.ts处理
5. **叙事生成**：政策效果通过narrator.ts生成史册记录
6. **长期影响**：政策持续时间结束后，可能留下永久性影响
